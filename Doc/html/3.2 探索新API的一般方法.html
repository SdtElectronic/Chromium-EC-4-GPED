这还用你教，给我API文档的页面

  • EC采用Linux kernel式的coding style，这意味着EC没有独立的文档，源码的注释就是文档
  • 尽管我们把注释名义上称为文档，实际上它并不能当成一般意义上的文档来对待。首先源码的目录结构显然不是为了方便文档阅读而设计的，API相关的文件分散在大量无关的其他依赖文件中；而API源码文件中说明API使用方法的注释混杂在其他解释代码的注释中
